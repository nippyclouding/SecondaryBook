## 변경 이유

### Spring Batch 5 마이그레이션
현재 mvc의 BatchConfig는 Spring Batch 4.x 스타일 (JobBuilderFactory, StepBuilderFactory).
batch 서버는 Spring Boot 3.3.4 → Spring Batch 5.x 사용.
- `JobBuilderFactory` → `JobBuilder(name, jobRepository)`
- `StepBuilderFactory` → `StepBuilder(name, jobRepository)`

### ShedLock 제거
배치 서버는 단독 EC2 1대 → 중복 실행 자체가 불가 → ShedLock 불필요.
- batch: SchedulerConfig에서 @EnableSchedulerLock, LockProvider 제거
- mvc: ShedLock 의존성 제거, SchedulerConfig 단순화

### 최소 의존성 원칙
- `SettlementService` 전체 복사 대신 `BatchSettlementService`로 필요한 메서드만 추출
- `MailService` 전체 복사 대신 `BatchMailService`로 메일 발송 메서드만 추출 (Redis 의존성 제거)
- `TradeMapper` 전체 복사 대신 필요한 2개 메서드만 선언

## 실행 순서

### 1단계: batch 디렉토리 구조 생성
```
spring/batch/
├── job/settlement/         ← 배치 Job 파일들
├── scheduler/              ← 스케줄러
├── settlement/             ← 정산 도메인
├── trade/                  ← 거래 도메인 (최소)
├── member/                 ← 회원 도메인 (최소)
├── util/                   ← 유틸리티
└── config/                 ← 설정
```

### 2단계: 도메인 파일 생성 (공유 코드)
- SettlementMapper, SettlementVO, SettlementStatus
- BatchSettlementService (markAsInsufficient만)
- MemberVO, MemberMapper (findByMemberSeq만)
- BatchMailService (sendInsufficientBalanceEmail만)
- TradeMapper (2개 메서드만)
- AesEncryptionUtil

### 3단계: 배치 Job 파일 이동
- batch/* 파일 → spring/batch/job/settlement/
- 패키지명 변경: project.batch → spring.batch.job.settlement

### 4단계: 스케줄러 파일 이동
- SettlementScheduler → spring/batch/scheduler/
- SafePaymentScheduler → spring/batch/scheduler/
- @SchedulerLock 제거

### 5단계: BatchConfig 재작성 (Spring Batch 5)
- JobBuilderFactory → JobBuilder
- StepBuilderFactory → StepBuilder

### 6단계: SchedulerConfig 생성 (batch용)
- @EnableScheduling만 (ShedLock 없음)

### 7단계: Mapper XML 복사 (batch resources)
- settlementMapper.xml 전체
- tradeMapper.xml (2개 쿼리만)
- memberMapper.xml (1개 쿼리만)

### 8단계: application.properties 업데이트
- aes.secret-key 추가
- mail 설정 추가
- spring.batch.jdbc.initialize-schema 설정

### 9단계: mvc 정리
- batch 관련 파일 삭제 (이동한 파일들)
- SchedulerConfig 단순화 (ShedLock 제거)
- pom.xml에서 spring-batch-core, shedlock 의존성 제거

### 10단계: 테스트 파일 이동
- project/batch/*Test.java → spring/batch/job/settlement/
